Power Market Load & Generation Visualization

This project uses MISO South market data to look at sytem load, forcast load, wind generation, and solar generation. I mainly wanted to se how load changes over time and if the renewable numbers show any pattern.

This notebook is kept simpe because the assignment is about clear visuals not building a full application. MISO is an Independent System Operator that manages the grid for a secotion of North America more info can be found at https://www.misoenergy.org/ .

Connecting to Course Goals

This project helped me practice using the Altair visualization software. I have used other tools in the past, such as Matplotlib, but I was not aware of Altair before this course. I used Altair to build several charts, including forecasted load, wind, and solar data, with each chart answering a different question. I compared different chart types, such as line charts and scatter plots, to show different patterns in the data and support different conclusions. I also gathered feedback from three people to evaluate how clear the visuals were and how they could be improved to better help users answer the intended questions.

Module 1: Finding your Data

Data Source

  • The dataset in the notebook is MISO South hourly data that was cleaned the csv was loaded directly into the initial notebook.

Key Columns

  • timestamp for the hour
  • actual_load_mw for actual load
  • forecast_load_mw for forecast load
  • wind_generation_mw
  • solar_generation_mw
  • total_renewable_generation_mw
  • region

Main Objective

  • See the overall load pattern
  • Compare MISO actual load vs forecast load
  • Observe the wind and solar output over time

Questions I wanted to answer

  • What day and hour has the highest and lowest load?
  • Is the forecast usually close to the actual load?
  • Does solar follow a more obvious daily pattern than wind?
  • Does renewable generation seem to move with load at all?

Visuals I looked at for ideas

  • When I looked at examples of energy and market dashboards, I noticed that load is usually shown with line charts over time. That makes sense because load is something that changes hour by hour, so a line chart is one of the clearest ways to show the pattern.
  • I also saw that actual load and forecast load are often put on the same chart using two lines. I think that works well because it makes it easy to see where the forecast was close and where it missed.
  • For relationships between variables, scatter plots are pretty common. In this case, a scatter plot can help show whether renewable generation seems to move with load or not.
  • One thing I noticed in a lot of real dashboards is that they try to show too much information at once. That may be useful for operations, but it can also make the visuals harder to read quickly. For my project, I wanted something more simple and easier to explain, so I focused on a small number of charts that match my main questions.

Module 2: Tasks and Low-Fidelity Prototype

The project is suppose to answer three main things.

Task 1: Look at the overall load pattern

The first task is to understand how load changes over time. I care about this because load is one of the main signals in power market data, and I want to see when it is high, when it drops, and whether there are repeated daily patterns.

The main way to do this is with a line chart of actual_load_mw over timestamp. This task is meant to show high and low periods, general trends, and repeated movement by hour or by day. It uses the actual load data directly, and it would usually be the first step in the workflow because it gives the basic picture before comparing anything else.

The person doing this task would probably be a student, analyst, trader or someone trying to understand market conditions at a high level.

Task 2: Compare actual load and forecast load

The second task is to compare actual load with forecast load. I want to do this because it helps show whether the forecast was close to what really happened, which is useful for understanding how well expectations matched reality.

The easiest way to do this is to put both series on the same chart using two lines. This makes it easier to notice where the lines overlap and where they separate. The task focuses on actual_load_mw, forecast_load_mw, and timestamp. I would do this after first looking at the actual load by itself, since it builds on that first view.

This task fits someone doing load analysis or looking at forecast performance.

Task 3: Look at wind and solar behavior

The third task is to look at wind and solar generation and see how they behave over time. I wanted this part because renewable output adds useful context and might help explain some of the load patterns that are happening today.

To do this, I would use a line chart for wind_generation_mw and solar_generation_mw over time. I could also use a scatter plot to compare total_renewable_generation_mw with actual_load_mw if I want to see whether there is any visible relationship between the two. This task looks for things like solar’s daytime pattern, wind variation, and whether renewables seem connected to load at all. I would do this after the load charts so I already have an idea of the system picture in mind.

This task would make sense for a student or power market analyst trying to understand renewable trends.

Design Justification

  • I used line chart because most of the data is changing hour by hour.
  • A line chart makes actual load and forecasted load easy to compare.
  • I used a scatterplot for load vs renewable generation because it was a quick way to see if there were any correlations.
  • I did not want too many visuals so kept it to a few charts that were easy to read.

Findings and Future Improvements

I think overall the line charts worked very well because most of the data changed hour by hour. This makd it easier to see the load forecast, wind and solar patterns clearly. The actual vs forecast shows us where the forecast was close and where it missed target.

For future impprovements, I would keep the lighter background, improve the hover detail as it could use some work and attempt to add better zoom options to get a more granular look at the charts. I would also work or try using different charts other than line and scatter plots to see if they can convey the information better.
